# CLAUDE.md

## 维特根斯坦纪律（防落入形式主义陷阱）

1. **TLC 验证结果 = 模型内部一致性，≠ 方法论功效证据。** 文档和注释中涉及 `<>[]StrongEnough` 等性质时，必须标注前提假设（单调性、无遗忘等），禁止将其表述为"方法有效"的证明。
2. **每引入一个简化假设，必须在 `docs/constraints-and-roadmap.md` 同步登记，标注它对已验证性质的影响。** 不登记的简化 = 伪精确性（Wittgenstein 意义上的 bewitchment by language）。
3. **区分"模型的真"和"世界的真"：** 规范的正确性归 TLC，方法论的有效性归实证——两者不互相替代。永远不说"TLA+ 证明了思维树方法论可行"，只说"TLA+ 验证了该规范的 type safety 和 liveness 在给定假设下成立"。
